# 前端集成说明

## 1. 目录结构（需部署到服务器）

```
日报预测/
├── data/processed/weekly_daily.csv    ← 周级数据（必须）
├── data/processed/weekly_gantt.csv    ← 甘特图基准（可选，无则跳过基准）
├── models/forecast/*.pkl              ← 训练好的模型（必须）
├── models/forecast/feature_columns.txt ← 特征列表（必须）
├── src/
│   ├── predict_api.py                 ← 预测 API
│   └── benchmark.py                   ← 基准模块
└── requirements.txt
```

## 2. 环境

```bash
pip install -r requirements.txt
```

依赖：`numpy>=1.24, pandas>=2.0, scikit-learn>=1.3, xgboost>=2.0, joblib>=1.3`

## 4. 前端输入

前端只需要提供一个下拉框让用户选择项目（P001/P003），其他全自动。

调用方式（后端）：

```python
from src.predict_api import predict

result = predict("P003")  # 返回 dict，直接可序列化
```

## 5. 数据更新流程

当有新日报进来时，需要重新跑：

```bash
# 1. 把新日报TXT放到 data/daily/P00X/ 下
# 2. 重新生成周级数据
python -c "from src.process_daily import process; process('data/daily','data/processed')"
# 3. 重新训练
python src/train_forecast.py
```

之后再次调用 `predict_api.py` 预测。

## 6. 前端展示建议

| 板块 | 建议展示方式 |
|------|-------------|
| 人员 | 数值 + 区间进度条（low-high 范围） |
| 机械 | 列表，标 `[规则]` 的灰色显示 |
| 活动 | 柱状图（土建/钢结构/设备安装/装修） |
| 基准 | 仪表盘（67%位置、平均69%、评分文字） |
| 天气 | 图标 + 天数 |
