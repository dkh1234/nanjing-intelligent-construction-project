# 施工活动预测难点分析

## 当前能力与局限

### 可以可靠预测

| 目标 | 精度 | 原因 |
|------|------|------|
| 日均施工人员 | MAE ~95人 | 连续惯性量，周间变化缓慢，两个项目模式相近 |
| 最大施工人员 | MAE ~73人 | 同上 |
| 塔吊数量 | MAE ~0 | 项目存续期间基本不变 |
| 装载机数量 | MAE ~0 | 同上 |
| 挖机数量 | MAE ~0.3台 | 变化范围小，模式稳定走 |
| 工序频次总量 | 趋势正确但数值偏低 | 能判断上升/下降，绝对值不准 |

### 无法可靠预测

- 下周具体哪些区域/子项会施工（窑尾还是水泥库？）
- 具体施工内容是安装还是拆除还是浇筑
- 哪些任务会在下周开始、哪些会结束
- 各工序的绝对频次（土建/钢结构/设备安装的具体次数）

---

## 难点一：训练数据量不足以学习活动模式

### 现状

- P001：49 周
- P003：61 周
- 合计：约 108 条周级配对

### 问题

两个项目处于不同阶段、不同节奏：

```
P001 第30周: 土建120次  钢结构80次  设备安装60次
P003 第30周: 土建5次    钢结构0次   设备安装2次
```

同周次的活动量差距 20-60 倍。模型无法从中提取"在第 N 周通常该干什么"的通用规律。

### 需要的条件

至少 **10+ 个同类型项目**的完整日报数据，覆盖从开工到完工的全周期，模型才能学到可靠的活动演进模式。

---

## 难点二：工序分类太粗，丢失了内容信息

### 现状

所有施工条目被归为 4 类：土建、钢结构、设备安装、装修。

### 问题

同一分类下包含完全不同的具体工作：

**设备安装** 这个标签实际对应 20+ 种工作：
- 烧成窑尾风管支模浇筑
- 水泥磨出料罩组对
- 包装机安装
- 篦冷机人孔门焊接
- V型选粉机安装
- 辊压机砂浆墩制作
- 取料机轨道找正
- 太极锥组对
- ...

从数字上无法区分这是在做"收尾阶段的零星安装"还是"主体安装高峰"。

### 可能的改进方向

将工序分类从 4 类扩展到按子项分类，例如：

- 烧成窑尾施工
- 烧成窑头施工
- 原料粉磨施工
- 水泥粉磨施工
- 水泥储存施工
- 煤粉制备施工
- 辅料堆场施工
- ...

这需要从日报原文中提取子项名称作为分类依据。

---

## 难点三：相同的活动数字可以对应完全不同的施工阶段

### 示例

```
P003 第58周: 设备安装 18 次 ── 主要是收尘灰斗焊接、轨道找正
P003 第59周: 设备安装 99 次 ── 突然新增风管支模、非标制作、太极锥组对
```

### 问题

数字跳了 5 倍，不是模型预测的偏差，而是**施工内容发生了质变**——从"收尾工作"切换到了"主体安装高峰"。这种质变信号藏在数字背后的施工描述里，模型只看数字看不到。

### 推论

预测"设备安装 88 次"的价值有限——项目经理更关心的是数字变化背后的**施工内容切换**，而不是绝对数字本身。

---

## 难点四：工程内容不可从纯数据反推

### 核心问题

下周具体做什么，取决于：

- 施工组织设计
- 图纸交付进度
- 设备到货计划
- 甲供材到场情况
- 各工序的逻辑依赖关系（如先浇筑基础才能安装设备）

这些信息**不在日报里，也不在甘特图里**。模型没有这些输入，就不可能准确预测具体施工内容。

### 推论

机器学习模型在现有数据条件下，定位应该放在**补充参考**而非**替代施工计划**。预测的价值在于：
- 人员/机械的量化预判（帮物资准备）
- 活动总量的趋势判断（帮进度管理）
- 与历史项目的对比基准（帮进度评估）

---

## 结论

| 层次 | 能不能 | 前提条件 |
|------|--------|---------|
| 下周需要多少人 | 能（MAE ~10%） | 2个项目足够 |
| 下周用多少机械 | 能（MAE ~0-0.3台） | 恒定设备直接用规则 |
| 下周做多少活 | 部分能（趋势方向对） | 需核实当前处于什么阶段 |
| 下周在哪个区域做 | **不能** | 需要子项级别的活动分类 |
| 下周具体做什么工序 | **不能** | 需要 10+ 同类型项目 + 更细分类 + 施工逻辑约束 |
| 当前进度是快是慢 | 能（S曲线基准） | 甘特图已有，已实现 |

## 短期可改进项

1. 用甘特图做 **"同阶段任务推荐"**：给定当前周次和工序分布，从 5 个已完工项目中找出相似阶段，列出当时在做哪些类型的任务
2. **工序分类细化**：从当前 4 大类扩展到按子项分类（需要从日报文本中提取子项名称）
3. **人员预测加置信区间**：输出"日均 978 人（区间 850-1100）"而非点值

## 长期方向

积累 10+ 个同类型项目的完整日报后，活动预测的精度会有质的提升。在此之前，"下周需要多少资源"比"下周要做什么"更适合作为模型的输出目标。
